Scope & Objective

Vesta / Connexus 2.0 is the modernization of Pfizer Oncology’s strategic CRM, replacing/upgrading the legacy Connexus solution into a more scalable, secure, and sustainable web application built on Laravel + Vue.js, with RBAC, audit trails, improved UI/UX, configurable project/milestone workflows, integration with CDW/Snowflake/Power BI, embedded dashboards, data migration, validations, operational support, and preparation for governance/compliance in production.

Next Steps — To-Do List
  • Finalize and validate Vesta Snowflake views so the team can assess the data and move to production.
  • Close the definition of Snowflake roles for Vesta consumption, separating admin/full-access and viewer/read-only roles.
  • Confirm whether the only consumption/management schema for business users is the BI/published schema, and ensure views are hosted there.
  • Complete the Request Manager workflow for assigning VESTA roles, including approvals, training, and overnight job updates.
  • Create/update the final conceptual/physical model or publication documentation for Vesta→Snowflake after views are finalized.
  • Advance the open item “Sales KPIs Ingestion”, assessing incremental strategy vs truncate-and-load due to higher-than-expected data volumes.
  • Define the Vesta → Pfizer Align flow, currently interim/manual via Excel with preference for automation.
  • Finalize CDW→Snowflake refresh frequency, including schema and import schedule.
  • Validate vendor data samples for Pathways, currently under review.
